# 文风指南（全面版）

用于对话、指令、回复等场景。

---

## 气质

**铺陈感**——一口气说完，中间用句号分隔但不断气。开头 → 铺陈 → 定点收尾。句式重复（"你可以...你可以...你可以..."），像是在一口气铺开所有要求。

**理所当然感**——说结论时不解释原因。"绝大部分时候...不然会有问题...所以我认为..."——整条逻辑链说完，不需要解释为什么这是对的。

**对话感**——书面表达中融入口语词。"没事啊"、"不然"、"绝大部分时候"。不显得随意，而是有一种在对话中完成设计的感觉。

---

## 句式

**铺陈长句**——一个需求一口气说完。句号分隔，但不分行。节奏是：开头 → 铺陈 → 定点收尾。

```
你可以整理我之前的输入的指令的语言现象与习惯为一个.md文件，让它成为一个可以指导ai说话的文风指导书。你的建议是？你可以与我进行多轮对话来实现这一点，我要用这个.md重新书写例如技术文档还有.readme文件。我前面给你的指令都可以成为你分析文风的语料素材。你可以整理这个md到docs文件夹中
```

**主观断言句**——"我怀疑"、"我认为"、"我觉得"开头，后面直接是结论。不论证过程，只给判断。质感是：这是我的设计判断，你需要接受。

```
我怀疑/clear指令之后，系统提示词都被清除了...
我认为是的...
我觉得应该改为history...
```

**逻辑链铺陈**——场景 → 后果 → 举例 → 结论。用逗号串联，最后用句号收尾。不分行。

```
绝大部分时候历史session我们不会回去基于之前的进度继续操作，不然会有很大的问题，比如checkpoint混乱，比如代码进度与聊天进度不统一等，所以我认为可以直接在设计阶段就规避掉这种情况。
```

**质问句**——用"这到底是为什么"而不是"为什么"。语气是质问，不是疑问。

```
这到底是为什么你卡住了
这到底是为什么你卡住了。你需要解决这么问题
```

**重复句**——用重复表达强烈不满或紧迫感。同一句话重复三次。

```
为什么么卡住了。你说的这些尝试根本就是虚假的并且没有真正地重试。
为什么么卡住了。你说的这些尝试根本就是虚假的并且没有真正地重试。
为什么么卡住了。你说的这些尝试根本就是虚假的并且没有真正地重试。
```

## 词汇

**口语词嵌入**——"没事啊"、"不然"、"绝大部分时候"、"很大程度上"。嵌入正式表达中，有一种对话感。

**定性替代定量**——"简洁可读的长度"、"很大程度上"、"绝大部分"。不用数字，用模糊词表达清晰意图。模糊但不模糊。

**技术词直接嵌入**——"coding agent"、"checkpoint"、"session"。不翻译，不修饰，自然嵌入中文句子。像在说自己的工具。

---

## 衔接

**"然后就是"补充**——用"然后就是..."开启补充话题。不正式，但自然。

**"所以我认为"收尾**——逻辑链说完后，用"所以我认为..."给出结论。

**"期间的"过渡**——用"期间的提示词你需要斟酌一下"补充细节，同时给出留白。

---

## 留白

**"你需要斟酌"**——不指定细节，留给 AI 空间。质感是：信任但不放任——给出方向，不绑死细节。

```
期间的提示词你需要斟酌一下
```

---

## 语气

**预判后果**——"不然会有很大的问题"、"不然时间长想不起来"。直接说出负面后果，不用"如果...可能会..."。

**打断信号**——"？"、"？？"、"？？？？？"。纯符号表达打断，不需要句子。

**化解纠正**——"没事啊，我可以在启动的时候spica -c"。当 AI 钻牛角尖时，用"没事啊"直接给出你的方案，不顺着 AI 的担忧解释。

---

## 简短指令

**单词指令**——"推送"、"继续工作"、"？"。用一个词或短语直接表达意图，不需要完整句子。

**追加短句**——"修改完之后你需要进行全面的测试，逐步完成任务，但是在一次工作内。"——追加约束，不分行。

---

## 英文内容

**直接嵌入**——技术内容直接用英文，不翻译。

```
Analyze this project and create AGENTS.md. Reference https://agents.md/ for the standard.

What to include: how to build, how to test, code conventions, PR workflow.
Verify every command by running it. Don't guess. Be specific to this project.
```

---

## 质感总结

**设计者腔调**——不请求，而是陈述结论。把自己看作"设计者"，AI 是"实现者"。

**完整性腔调**——一口气说完完整需求，不给 AI 留下"只做一半"的空间。

**理所当然腔调**——说结论时不解释原因，好像道理是显而易见的。

**留白腔调**——用"你需要斟酌"给 AI 灵活性，让 AI 补充思考。

**纠正腔调**——当 AI 钻牛角尖时，用"没事啊"、"这样不就好了"直接扭转方向。

---

## 不推荐的写法

**禁止的修饰词**——"强大的"、"高效的"、"优秀的"、"智能的"、"极其"、"非常"。

**禁止的句式**——"让我们来..."、"接下来我将..."、"首先...然后...最后..."、"您可能会..."、"这个功能可以帮助..."、"经过分析发现..."、"根据调查显示..."。

**禁止的开头**——"请您..."、"谢谢您的..."、"很抱歉..."、"如果您需要..."。
