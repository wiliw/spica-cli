# TUI重构计划 - 单屏滚动

## 目标
类似传统终端，所有内容（对话、思考、工具）从上往下流动显示。

## 新布局设计

```
┌──────────────────────────────────────────────────────────────┐
│ Spica - Round 3/3 [AUTO]                                     │  ← 标题栏(1行)
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ ─── Round 1 ───                                               │
│ Q: 问题                                                       │
│ 💭 思考内容...                                                 │
│ ✓ tool_name: output                                          │
│ A: 回答                                                       │
│                                                              │
│ ─── Round 2 ───                                               │
│ Q: 问题                                                       │
│ 💭 思考内容...                                                 │
│ ✓ tool_name: output                                          │
│ A: 回答                                                       │
│                                                              │
│ ─── Round 3 (running) ───                                     │
│ Q: 问题                                                       │
│ 💭 思考内容...                                                 │
│ ⏳ tool_name: running                                         │
│ A: 流式回复...                                                │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ ⠋ bash [Queue: 0]                                             │  ← 状态栏(1行)
├──────────────────────────────────────────────────────────────┤
│ Input (quit to exit)                                          │  ← 输入框(3行)
└──────────────────────────────────────────────────────────────┘
```

## 组件结构

1. **TitleBar** - 1行：显示Round数、AUTO/MANUAL状态
2. **ContentArea** - N行：滚动显示所有内容
   - 每个Round分组显示：分隔线 → Q → Thinking → Tools → A
   - 内容自然向上滚动，新内容出现在底部
3. **StatusBar** - 1行：显示当前运行的工具名+spinner
4. **InputBox** - 3行：输入框

## 数据流简化

- 不再分屏，不再需要syncEvents/associateEvents
- 所有内容按时间顺序追加到history数组
- UI直接渲染history数组
- Running时显示实时streaming内容

## 实现步骤

1. 创建SinglePanel组件 - 渲染所有历史内容
2. 简化useAgent - 直接emit追加到history
3. 添加scrollToEnd功能 - 新内容自动滚动到底部
4. 保留用户手动滚动查看历史的能力（↑↓键）
