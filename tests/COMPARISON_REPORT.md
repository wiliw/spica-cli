# AGENTS.md 对比测试报告

## 测试日期
2026-06-02

## 测试目的
对比完整版 AGENTS.md（182行）vs 精简版（35行）对 AI 理解性能的影响

## 测试版本
| 版本 | 行数 | 特点 |
|------|------|------|
| 完整版 | 182行 | 详细架构、工具列表、测试文件、平台兼容性 |
| 精简版 | 35行 | 核心信息压缩：架构、命令、风格 |

## 测试任务
1. 项目语言识别（简单）
2. bash 工具模式理解（中等）
3. 测试命令识别（简单）

| 任务 | 耗时 |

| **总计** | **72s** |

---

## 分析结论

### 关键发现
1. **响应时间几乎相同**: 总耗时差异仅 1秒（73s vs 72s）
2. **任务类型影响更大**: 
   - 简单问题（语言识别）两版本耗时相近
   - 中等问题（bash模式）精简版略慢
3. **Token消耗未捕获**: spica输出格式变化，需要改进解析

### 意外发现
精简版在某些任务上反而更快（任务3: 24s vs 30s），可能因为：
- AI 更快定位关键信息（文档短）
- 减少"噪音"信息干扰

### 局限性
1. 测试任务偏简单，未覆盖复杂架构理解
2. Token数据未捕获，无法评估上下文效率
3. 样本量小（仅3个任务）

## 建议

### 推荐策略: 分层式 AGENTS.md
```markdown
# AGENTS.md (~50行)

## Project
CLI 工具，AI 编程助手，TypeScript + Node.js 18+

## Commands
npm run dev | build | test:run

## Architecture
- src/agent.ts: 核心 Agent
- src/tools/: 25 工具
- src/cli/ui/: TUI 界面

## Code Style
无注释，TypeScript strict，工具返回 { success, output?, error? }

## Details
详细文档见: docs/ARCHITECTURE.md, docs/TOOLS.md
```

### 优势
1. **平衡**: 核心信息齐全，长度适中
2. **按需**: AI 可用 file_read 获取详细文档
3. **Token效率**: 比182行节省 ~70% 上下文
