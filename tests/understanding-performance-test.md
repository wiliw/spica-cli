# 理解性能测试方案

## 目的
测试 AI agent（spica）对项目的理解程度和效率

## 测试维度

### 1. 架构理解
测试 agent 是否理解项目核心架构

**测试任务：**
- 问："这个项目的核心架构是什么？"
- 问："agent.ts 的主要职责是什么？"
- 问："工具系统是如何组织的？"

**评估指标：**
- 答案准确度（人工评分 1-5）
- 回答时间（秒）
- Token 使用量

### 2. 功能理解
测试 agent 是否理解各工具的功能

**测试任务：**
- 问："bash 工具有哪些模式？"
- 问："file_edit 和 file_multi_edit 有什么区别？"
- 问："git 工具支持哪些操作？"

**评估指标：**
- 答案准确度
- 是否遗漏关键功能
- 回答完整性

### 3. 代码风格理解
测试 agent 是否理解项目的代码风格

**测试任务：**
- 让 agent 写一个新工具
- 检查是否符合项目风格：
  - 无注释
  - TypeScript strict
  - 简洁代码
  - 符合命名规范

**评估指标：**
- 风格符合度（%）
- 需要修改次数

### 4. 命令理解
测试 agent 是否知道关键命令

**测试任务：**
- 问："如何启动开发模式？"
- 问："如何运行测试？"
- 问："如何构建项目？"

**评估指标：**
- 命令正确性
- 是否知道参数

### 5. 上下文理解
测试 agent 是否理解项目上下文

**测试任务：**
- 给 agent 一个新需求
- 检查是否：
  - 知道修改哪个文件
  - 知道如何测试
  - 知道依赖关系

**评估指标：**
- 定位准确度
- 是否探索相关文件
- Token 使用效率

### 6. 错误处理理解
测试 agent 是否理解错误处理机制

**测试任务：**
- 故意触发一个错误
- 检查 agent 如何处理：
  - 是否理解错误类型
  - 是否知道修复方法
  - 是否知道如何测试修复

**评估指标：**
- 错误诊断准确度
- 修复效率

### 7. 项目导航
测试 agent 探索项目的效率

**测试任务：**
- 问："找出所有与 skills 相关的文件"
- 问："找出 ProcessMonitor 的实现"
- 问："找出所有测试文件"

**评估指标：**
- 导航时间
- 是否找到所有相关文件
- Token 使用量

## 测试执行方法

### 方法 1：人工评估
```
1. 启动 spica
2. 执行测试任务
3. 人工评分
4. 记录 token 使用和时间
```

### 方法 2：自动化脚本
```bash
# 创建测试脚本
for task in tasks; do
  echo "Task: $task"
  start_time=$(date +%s)
  spica run "$task"
  end_time=$(date +%s)
  echo "Time: $((end_time - start_time))s"
done
```

### 方法 3：对比测试
```
对比不同 AGENTS.md 版本：
- 当前复杂版本（182行）
- 精简版本（50行）
- 基于测试版本（30行）

运行相同测试任务，对比：
- 理解准确度
- Token 使用
- 响应时间
```
